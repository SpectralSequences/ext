A TokenBucket is a bucket containing a fixed number of "tokens". Threads can take request to take out a token. If no tokens are available, the thread is blocked (while consuming no CPU time) until a token is released and returned to the bucket. When this happens, one of the threads waiting for a token is woken up and takes the token. There is no guarantee as to which thread in the queue gets woken up when a token is available.

A TokenBucket is useful for limiting the number of active threads used by a function/program.
